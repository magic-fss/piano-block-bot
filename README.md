## ✨ 功能特性

- **超极速检测**：合并截图 + NumPy 布尔掩码，跳过 OpenCV 冗余遍历，单循环极限响应
- **可视化区域**：4 个可拖拽的半透明检测框，实时覆盖到游戏对应列
- **实时日志**：运行中实时显示各区域累计点击数，暂停/停止自动输出统计报表
- **热键控制**：F10 启动/暂停、F12 停止、ESC 紧急退出，无需切换窗口
- **独立点击位置**：检测区域与点击位置解耦，缩小检测高度提速时不影响落点

## 🚀 使用方法

### 1. 启动游戏

打开 **钢琴块2**（窗口标题必须为 `钢琴块2`），进入任意关卡。
注：也可以修改源码里的"钢琴块2"为其它的名称

### 2. 运行脚本

```bash
python "piano_block_tui.py"
```

> ⚠️ **必须以管理员身份运行**，否则全局热键（F10/F12/ESC）无法注册。

### 3. 调整检测区域

脚本启动后会弹出 4 个带彩色边框的半透明窗口：

| 颜色 | 对应列 |
|---|---|
| 🔴 红色 | 第 1 列 |
| 🟢 绿色 | 第 2 列 |
| 🔵 蓝色 | 第 3 列 |
| 🟡 黄色 | 第 4 列 |

**拖拽调整**：用鼠标按住任意检测框内部，拖动到游戏对应列的黑块下落区域。

**调整原则**：
- 框的**底部**对齐游戏底部黑块落点
- 框的**宽度**略大于单列宽度
- 框的**高度**越小检测越快（默认 100px，高速场景可改 60~80）

### 4. 开始自动点击

按 **`F10`** 启动，脚本开始自动检测黑块并点击。

## ⚙️ 配置说明

在脚本顶部「终极速度配置」区修改以下参数：

```python
# 区域尺寸
AREA_WIDTH  = 180   # 检测框宽度（像素）
AREA_HEIGHT = 100   # 检测框高度（像素），越小越快
AREA_GAP    = 20    # 列间距（像素）

# 黑块检测阈值
GRAY_TOLERANCE  = 6   # 灰度容忍度（0~255），越小越严格
MIN_BLACK_PIXELS = 50  # 触发点击的最小黑像素数

# 响应延迟
LOOP_DELAY = 0   # 0 = 极限响应；如需降低CPU占用可改 0.001
```

### 高速场景调参建议

| 场景 | AREA_HEIGHT | GRAY_TOLERANCE | MIN_BLACK_PIXELS |
|---|---|---|---|
| 低速/入门 | 150~200 | 6 | 50 |
| 中速/普通 | 100~120 | 6 | 50 |
| 高速/困难 | 60~80 | 8~10 | 30~40 |

> 降低 `AREA_HEIGHT` 是提速最有效的手段，因为直接减少了每次截图的数据量。

## ⚠️ 注意事项

1. **管理员权限**：`keyboard` 库注册全局热键需要管理员权限，否则热键无效。
2. **关闭鼠标加速**：控制面板 → 鼠标 → 指针选项 → **取消勾选「提高指针精确度」**，避免 Windows 鼠标加速干扰点击精度。
3. **窗口标题匹配**：脚本通过窗口标题 `"钢琴块2"` 定位游戏，请确保游戏窗口标题一致。
4. **单显示器**：当前版本基于绝对屏幕坐标，多显示器环境下请确保游戏在主显示器运行。
5. **防检测**：本工具仅用于个人娱乐学习，请勿在联网排行榜或竞技场景中使用。

## 🛠️ 技术栈

- **截图**：`mss`（跨平台极速截图）
- **图像处理**：`numpy` + `OpenCV`（灰度/阈值/像素统计）
- **窗口操作**：`pygetwindow` + `pywin32`
- **热键监听**：`keyboard`（全局热键）
- **GUI**：`tkinter`（可视化拖拽区域）
